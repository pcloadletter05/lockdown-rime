**Taskbar object/module with these responsibilities:**

**1. Taskbar DOM construction** (called from `init()`):
Build the taskbar inside `#desktop` (NOT document.body) using `document.createElement()`:
```

  button#start-button.raised

**2. Start button behavior:**
- Click toggles Start menu visibility
- While menu is open, Start button gets class `pressed` (remove `raised`, add `pressed`)
- While menu is closed, Start button gets class `raised` (remove `pressed`, add `raised`)
- Use mousedown (not click) for the toggle to prevent race conditions with the document dismiss handler

**3. Start menu construction** (built once, hidden by default with `display: none`):
```
div.start-menu.raised (display: none)
  div.start-menu-sidebar
    span.start-menu-sidebar-text "Windows NT"
  div.start-menu-items
    div.start-menu-item [icon] "Programs" [submenu arrow ">"]

**4. Start menu interaction:**
- Hovering over a menu item with a submenu shows the submenu (use mouseenter/mouseleave with a 200ms delay to prevent flicker)
- Clicking a menu item that has an app action: emit `EventBus.emit('app:launch', { appId: 'explorer' })` (or whichever appId), then close the Start menu
- Clicking "Shut Down..." shows the shutdown confirmation dialog (see below)
- Close menu by: clicking outside (use document `mousedown` handler), clicking Start button again, or clicking a menu item action
- When closing, also close all submenus

**5. Shut Down flow:**
- "Shut Down..." item opens an NT4 dialog (use existing `.nt4-dialog` CSS classes):
  - Title: "Shut Down Windows"
  - Body: "Are you sure you want to shut down?"
  - Buttons: "Yes" and "No" (use `.nt4-button` class)
  - Dialog z-index: 20000 (`.shutdown-dialog` class)
- "No" closes the dialog
- "Yes" removes everything from the page and shows the shutdown screen:
  - Full-screen black overlay (`.shutdown-overlay`) with z-index 30000
  - Orange text "It is now safe to turn off your computer." (`.shutdown-text`)
  - Uses `font-family: "Courier New", Courier, monospace` and `color: #FF8C00`

**6. System tray clock:**
- On init, call `updateClock()` which sets `#tray-clock` text to `TimeEngine.formatClockTime(TimeEngine.getMappedTime())`
- Set `setInterval(updateClock, 60000)` to update every minute
- Set `title` attribute on `#tray-clock` to `TimeEngine.formatFullDate(TimeEngine.getMappedTime())` -- updates on each tick
- Clock area (the `#system-tray` div) already has `.well` class for sunken bevel

**7. Window button management** (listen to EventBus events):
- `window:created`: Add a new button to `#taskbar-buttons`:
  ```
  button.taskbar-window-btn.raised
    span.taskbar-btn-icon [16x16 icon from event payload if available]
    span.taskbar-btn-title [title text]
  ```
  Store mapping of windowId to button element.
  Click handler on button:
    - If window is the active window AND not minimized: call `WindowManager.minimizeWindow(id)` (toggle behavior per user decision)
    - Else: if minimized, call `WindowManager.restoreWindow(id)`; if not minimized, call `WindowManager.focusWindow(id)`

- `window:focused`: Update all taskbar buttons -- remove `pressed` and add `raised` for all, then add `pressed` and remove `raised` for the focused window's button. If windowId is null, all buttons get `raised`.

- `window:minimized`: The button stays but gets `raised` class (not pressed).

- `window:restored`: The button gets `pressed` class (same as focused handling via the window:focused event that follows).

- `window:closed`: Remove the button from `#taskbar-buttons` and clean up the mapping.

**8. Initialization:**
At the bottom of the file, add a `DOMContentLoaded` listener that calls `Taskbar.init()`. The init function builds the taskbar DOM, Start menu, and starts the clock. It also sets up all EventBus listeners.

**AppRegistry object:**

Define a registry of available apps, each with: `id`, `title`, `icon` (inline SVG string), `defaultWidth`, `defaultHeight`. Phase 2 creates stub windows only -- Phase 3 fills them with real content.

```javascript
const AppRegistry = {
  apps: {
    'explorer': { title: 'Windows Explorer', icon: EXPLORER_SVG, width: 640, height: 480 },
    'outlook':  { title: 'Inbox - Outlook', icon: OUTLOOK_SVG, width: 700, height: 500 },
    'iexplore': { title: 'Microsoft Internet Explorer', icon: IE_SVG, width: 800, height: 600 },
    'notepad':  { title: 'Notepad', icon: NOTEPAD_SVG, width: 480, height: 360 },
    'mycomputer': { title: 'My Computer', icon: MYCOMPUTER_SVG, width: 500, height: 400 },
    'control-panel': { title: 'Control Panel', icon: SETTINGS_SVG, width: 450, height: 350 },
  },

  launch(appId, args) {
    const app = this.apps[appId];
    if (!app) return;
    WindowManager.createWindow({
      title: app.title,
      icon: app.icon,
      width: app.width,
      height: app.height,
      content: null  // Stub -- Phase 3 will provide content builders
    });
  }
};
```
